01. Context & Host Organization

My S7 **Expérience Professionnelle Internationale (EPI)** took place in **Malaysia**, focusing on mobile application development and edge AI integrations.

Organisational Scope:

Building a modern bioacoustics mobile client to classify local bird species entirely offline to support forest monitoring and citizen science.

Key Deliverables Specified:

  • ✔ 100% offline edge AI execution.
  • ✔ Cross-platform code using Flutter.
  • ✔ Local SQLite storage for observation history.
  • ✔ Robustness guards against silence and mic saturation.

02. Technical System Architecture

Audio Pipeline & Preprocessing

  • Resampling: Resampled on-device to 16,000 Hz Mono PCM 16-bit.
  • Energy Windowing: Automatic extraction of the most active 3-second segment.
  • Mel Spectrogram: Computed to output visual representations matching human auditory scale.

Mobile Client Stack

  • Framework: Flutter for modern, hardware-accelerated fluid UI.
  • ML Runtime: TensorFlow Lite (TFLite) using C Platform Channels.
  • Database: SQLite via Sqflite for persistent history.

03. Edge Deep Learning Models

We designed an **Ensemble Model** to combine architectural advantages, improving robustness against environmental noises.

// Validation Accuracies
• Compact CNN:75.74%
• MynaNet (Solo):80.16%
• Double Sigmoid Ensemble:81.41%
MynaNet (MobileNetV3 Small)

Lightweight inverted residuals with Squeeze-and-Excitation blocks to focus on specific audio frequency channels.

Why Sigmoid + BCE instead of Softmax?

Softmax forces mutual exclusion. Sigmoid handles silence, multi-label classifications, and environmental noise independently.

04. Edge AI Optimizations

Running deep neural networks offline on phones requires careful resource management.

  • •Float16 Quantization: Weights quantized from 32-bit to 16-bit, cutting model size in half and accelerating calculations.
  • •Warm-up Inference: Initiates a dummy classification on startup, removing a 500ms initial lag when the user first records.

DSP Guard Protections

Silence Guard: Blocks TFLite calls if audio RMS is below 0.005.
Clipping Guard: Warns user of microphone clipping saturation.
