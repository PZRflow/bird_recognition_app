// Problématique

Pourquoi Shazam échoue-t-il sur la nature ?

Shazam traditionnel repose sur l'extraction d'empreintes acoustiques (hashes) issues de morceaux de musique calmes et standardisés en studio.

Les bruits de la nature brisent cette logique :

  • Bruit de fond omniprésent (vent, pluie, insectes).
  • Variabilité biologique d'un même oiseau (dialectes).
  • Superposition acoustique (plusieurs oiseaux chantent en même temps).

La Solution : Remplacer le hachage par de l'intelligence artificielle (Deep Learning) capable de généraliser et d'abstraire les bruits parasites.

// Prétraitement DSP

De l'Audio Brut aux Spectrogrammes Mel

Pour qu'un réseau de neurones convolutif (conçu pour l'image) comprenne du son, nous devons convertir l'onde temporelle en une image temps-fréquence.

  • • Rééchantillonnage (FFmpeg) : Conversion du flux microphone à 16 000 Hz, Mono, PCM 16 bits. Indispensable pour la cohérence mathématique.
  • • Sélection d'Énergie : Découpe en fenêtres de 3 secondes. Le segment affichant la plus grande énergie brute est extrait pour l'inférence.
  • • Spectrogramme Mel : Transformation de Fourier (STFT) compressée selon l'échelle de Mel, qui mime la sensibilité de l'oreille humaine aux hauteurs de ton.
// Deep Learning

Deux Modèles de Départ Complémentaires

1. MynaNet (Réseau Principal)

Inspiré de MobileNetV3 Small (Inverted Residuals)

  • • Canaux de sortie : 64 Mel × 300 trames temporelles.
  • • Squeeze-and-Excitation : Recalcule dynamiquement le poids de chaque canal pour focaliser le réseau sur les fréquences du chant.
  • • Légèreté extrême : Moins de 3 millions de paramètres.
  • • Précision Solo : 80.16% de validation accuracy.

2. Compact CNN (Réseau Complémentaire)

Réseau Convolutif Standard à 4 Couches

  • • Canaux de sortie : 128 Mel × 128 trames temporelles.
  • • Global Average Pooling : Remplace les couches denses lourdes pour éviter le surapprentissage.
  • • Simplicité : Idéal pour capter les motifs grossiers et les chants lents.
  • • Précision Solo : 75.74% de validation accuracy.
// Choix Architecturaux Majeurs

Pourquoi la Sigmoïde plutôt que le Softmax ?

Le standard de l'industrie pour la classification d'images utilise une activation Softmax en sortie. Mais pour la bioacoustique de terrain, c'est une faiblesse majeure.

La contrainte Softmax :

Softmax force la somme de toutes les probabilités à être égale à 1.0. Face à un bruit inconnu ou du vent, le modèle est obligé d'attribuer une forte confiance à l'une de ses classes connues !

Le choix Sigmoïde + Binary Crossentropy (BCE) : chaque oiseau est évalué de manière indépendante entre 0 et 1.

  • Résistance au mimétisme (ex: le Drongo imitant le Tit-Babbler).
  • Possibilité de prédire "Inconnu" si tous les scores sont < 15%.
  • Détection naturelle multi-label.

Comparaison Mathématique

# Softmax (Exclusif)

\[P(y_i) = \frac{e^{z_i}}{\sum e^{z_j}}\]

Force une somme de 1. Confiance artificielle.

# Sigmoid (Indépendant)

\[P(y_i) = \frac{1}{1 + e^{-z_i}}\]

Évalue chaque espèce séparément. Idéal pour le bruit.

// Résultats

L'Ensemble Double Sigmoïde : Record de Précision

Compact CNN (Solo) 75.74%
MynaNet (Solo) 80.16%
Ensemble Double Sigmoïde (50/50) 81.41%

Pourquoi l'Ensemble surpasse les modèles solos ?

En moyennant les probabilités sigmoid des deux modèles (ratio 0.5 MynaNet + 0.5 CNN) :

  • • Réduction de la Variance : Les erreurs individuelles de chaque réseau s'annulent.
  • • Complémentarité Spectrale : MynaNet (64x300) excelle sur la dynamique temporelle, tandis que le CNN (128x128) excelle sur la précision fréquentielle.
// Déploiement Mobile

Edge AI & Optimisations Smartphones

Faire tourner de gros modèles de Deep Learning sur des puces de téléphones nécessite des techniques d'optimisation spécifiques.

  • • Quantification Float16 : Conversion des poids 32 bits en 16 bits sous TFLite. Divise le poids de l'APK par 2, accélère les calculs et limite la consommation thermique.
  • • Inférence de Préchauffage (Warm-up) : Lancement d'une fausse inférence (`_warmup()`) à l'initialisation de l'application. Évite un lag de 500ms au premier enregistrement réel de l'utilisateur.
  • • TTA par Fenêtre Glissante : Analyse de l'audio par blocs qui se chevauchent de 50%, retenant le score maximal pour une détection robuste.

// Robustesse Bioacoustique

Les DSP Guards : Sécuriser la Qualité d'Inférence

En forêt, l'utilisateur n'est pas ingénieur du son. L'application doit analyser intelligemment la qualité du microphone avant de faire confiance aux prédictions.

  • 1. Silence Guard : Si le signal RMS calculé sur le PCM brut est trop faible, l'application bloque l'inférence TFLite inutile et indique "Silence" pour préserver le processeur.
  • 2. Clipping Guard : Si le micro sature (téléphone trop proche de l'oiseau ou vent violent), l'application prévient l'utilisateur avec un message "Signal saturé" pour tempérer l'estimation.
